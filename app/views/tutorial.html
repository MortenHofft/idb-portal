iDigBio Specimen Portal Tutorial
(version 5.0, released June 2015)

Introduction

The iDigBio specimen portal (https://www.idigbio.org/portal) provides access to millions of records about neontological and paleontological specimens curated at museums and other institutions in the US. Records might include information about the specimen, when, where and by whom it was collected, the institution providing the data, and media related to the specimen (currently only images of the specimen or the specimen label are available). The main intent of this tutorial is to familiarize the user with the search capabilities of this portal.

Requirements

The iDigBio Portal requires the use of an HTML5 compliant browser with cookies enabled. The Portal has been tested succesfully against the following browsers:
Chrome
Firefox
Safari
Internet Explorer 10,11

Portal Navigation

The main features of the portal can be accessed through the Green Menu Bar (Fig. 1) near the top of the page. The menu choices include links to the iDigBio Home, the Portal Home page, the Search Records page, Tutorial (this page), and Feedback, which are detailed next.

The iDigBio Home link takes you away from the portal, and into the portion of iDigBio website containing information about the project, its workshops and working groups, research, education and outreach.

Use the Portal Home page to start searching Specimen Records quickly on a scientific name, and to view basic overall statistics about the existing data (i.e., distribution of data by collection type, and overall number of records). Search results are returned on a new page that allows a more comprehensive search.

Use the Search Records page for more comprehensive and specific searches. On this page, you can perform a Full Text Search and/or an Advanced Search and view the search results as detailed in the next section.

The Tutorial page is this page.

Use the Publishers page to view information about the datasets and publishers provided to iDigBio.

Use the Research Tools page to view research software that make use of services and data.

Use the Feedback page to provide input about what could be improved in the portal, technical problems you might be facing, questions on how to use the portal, or just to let us know that you are finding the portal useful!

Understanding and Performing Searches

In the Search Records page, Full Text Search (Fig. 2) allows you to search a term across all fields of the data simultaneously. If you search for ‘carex’, this term is searched in all fields, and the search returns records where ‘carex’ appears in a scientific name, genus, host, locality, or any other place where it appears, excluding substrings (parts of words). Searches are not case-sensitive, i.e., ‘carex’ and ‘Carex’ will return the same result. Checking the box only records with images restricts search results to those specimens that have related images.

The Advanced Search section of the page (Fig. 3), allows you to specify which fields to search and display (the Table view displays up to seven columns, depending on screen size). The default list of search fields includes: family, scientific name, genus, country, and state/province.

You can add additional fields by selecting the desired field in the Add a field box (Fig. 4), and reorder the fields by dragging the field box using the icon or the field name (only fields that have been indexed by iDigBio are presented in this list, the Specimen Records might have additional fields that are currently not indexed). To remove a field, click the x icon on the top right corner of each field box. The Sort by box (Fig. 5) gives a choice of fields on which to sort the results, while the Direction box (Fig. 6) allows you to switch between ordering the results in an ascending (a-to-z) or descending (z-to-a) manner. Alternatively, you can click the column that you would like sorted in the results; a second click changes the direction of the sorting.

The individual search boxes offer to autocomplete with a prediction of your search criteria that is based on data already in the system (Fig. 7). To return results, use the Search button (typing Enter allows you to enter more terms to be searched in the same field). Note that ‘Scientific Name’, ‘Genus’ and ‘Specific Epithet’ are three separate fields, and different providers opted to fill only ‘Scientific Name’, only ‘Genus’ and ‘Specific Epithet’, or both. Thus, you might need to perform multiple searches to find all specimens of a species.

The Family, Genus and Scientifc Name fields provide an "Add Synonyms" button (Fig. 7) which uses the Encyclopedia of Life API to find synonyms for user entered names. Clicking the button once you have entered a name or multiple names will add any found synonyms to the search field.

Present and Missing check boxes under each field refer to the presence or absence of data in the specified field for a record. For example, if you want to find specimens identified only to genus, you can use this feature, and the search would be genus (check box Present), specific epithet (check box Missing). (Fig. 8)

To restart or revise a search, use the Reset button (Fig. 10) or use the Clear button to simply clear the text boxes and check boxes on screen. To go back to a previous search, look up your history of queries under the Query drop-down box (Fig. 9). Not sure you can remember everything? The Tips & Hints link (Fig. 10) can give you short reminders.

Georeferenced Searches

Searching for georefenced specimens can be achieved by selecting the Latitude/Longitude Bounds field in the Add a field select box (Fig. 4). This field provides the ability to search within a rectangular set of bounds by supplying latitude and longitude points for the northwest and southeast corners of the bounding box (Fig. 11). The bounding corners display and use default global values for fields not filled. Searching for just the presence or absence of records with a geopoint can be achieved by using the present and missing checkboxes (Fig. 11).

Viewing Search Results

Current Results displays your current and previous searches, the number of records returned, and an option to download a CSV (comma separated value) file of the results. The formatted results can be displayed in Table view, Label view or the Images gallery. The Table view allows you to display up to seven columns (depending on your screen size). The Label view displays a stylized specimen label, with a pre-determined set of information (taxonomy, geography, and collection data) and a thumbnail of a related image when available. The Images gallery displays all images associated with the specimen records search results. Each image contains a pre-determined set of information (taxonomy, collection data) displayed as a label over the image. Clicking on the image will open its associated media record.

Clicking on a single row of the table or a single label presents the a complete specimen record page (Fig. 12), including all specimen data, any associated media if available and a map with the location where the specimen and similar specimens were collected (if georeferenced). Clicking on the image in the specimen page allows you to access the full-size image as well as metadata about the image. You can go back to the specimen information, using the Go To Specimen Record link, under the smaller version of the image.

The specimen record page also provides you with a view of the raw data in JSON format (Fig. 13) through the View Raw Data button. With the raw view you can have a peek at the data that is available to you or could be used to add value to other portals. It can also be used to view additional information created by iDigBio, such as the internal GUID or when it was ingested or updated in our system, and to detect if iDigBio is not accurately displaying the data as it came from the provider.

Georeferenced Results

Any records that have been georeferenced (i.e., latitude and longitude coordinates are present) are included in the small map visualization near the top of the search page. You can pan and zoom within this “maplet.” The button in the top right corner is a toggle between the maplet and a full-size version that fills your browser window. The visualization shows a heat map (Fig. 14) when the area displayed represents more than 5,000 records; the color of the square cells indicates density of specimen occurrences and corresponds to the legend. The size of the cells is dependent on your zoom level. Clustered points with record counts are displayed when there are less then 5,000 records and multipled points occur in or close to the same location on the map. Individual points are shown when the displayed area encompasses 5,000 records or less (Fig. 15). You can view summary data by clicking on individual points. Clicking on the taxon name in the summary pop-up will open a new page for the full specimen record. The full-size map has a second button in the top right that allows you to specify a zoom area by drawing a rectangle around your area of interest.

Downloading Search Results

Below the Query drop-down box (Fig. 9) is the results Download feature (Fig. 16). This feature allows you to obtain all the record data in the current search query in Darwin Core Archive (DwC-A) format. You must provide a valid email address in the Email input box in order to run the download. An email is required so a link to the DwC-A file can be sent to you should you run another query or navigate away from the Search Records page. The Approx. Download Time provides a rough calculation of the time it will take to generate the DwC-A. Once your download is ready, the time estimate is replaced with a link to the download.

A Darwin Core Archive is a zip file that includes one or more text CSV (comma separated value) files, each with a .txt extension and a descriptor (meta.xml) that contains metadata about the data set that you downloaded (e.g., who provided the data and the format). A data download that has images or other media associated with it will have two CSV files, one for specimen data and the other describing the media. The CSV file can be opened by any text editor or can be imported into a database or spreadsheet. A modestly sized download (100,000 records) can be readily opened in software like MS Excel. The easiest way to open a CSV file with Excel is to change the .txt extension to .csv before you open the file.
